# 纸品加工MES系统 V4.1 开发执行计划

> **定位**：严格对标 V4.1 传统MES架构，不含事件溯源、Schema引擎、ActionFlags等工业契约体系。可直接拆分为Jira任务，零架构错位。

---

## 一、总体策略

### 1.1 开发原则
- 先跑通主流程，再补齐复杂场景
- 所有规范分阶段生效，不一次性实现
- 后端优先建立"表结构 + 状态机 + API"
- 前端优先建立"页面骨架 + 固定UI布局"
- 复杂能力（快照、审计、统计报表）后置

### 1.2 四阶段交付模型

| 阶段    | 名称         | 核心目标                                     | 是否可联调 |
| ------- | ------------ | -------------------------------------------- | ---------- |
| Phase 1 | 系统骨架期   | 基础档案可用，加工单创建可跑                 | ❌          |
| Phase 2 | 主流程打通期 | 跑通完整单据闭环（创建→打印→回录→出库→结算） | ✅          |
| Phase 3 | 规则稳定期   | 业务规则固化、异常流程补齐、数据一致性保障   | ✅          |
| Phase 4 | 工程收口期   | 可上线、可验收、可抗异常                     | ✅          |

## 二、Phase 1：系统骨架期（第1-2周）

### 🎯目标
系统能跑起来，基础档案可维护，加工单可创建/查看，但业务不完整。

### 2.1 后端任务

#### P0-0：复卷重量分摊计算规格书（第1周前置，开工硬门槛）
- 输出 V4.1 文档 2.6.1《复卷重量分摊计算规格书》对应的可编码规格：英寸→mm换算、横截面积公式、单件重量分摊、修边分摊、尾差倒挤
- 至少 2 组完整用例（含手工计算期望值），作为 Phase 2/3 计费与回录的单元测试基准
- **此项未完成不得进入复卷相关编码**

#### P0-1：数据库建表（第1周前2天）
- 按 V4.1 第三章完整建表（3.3.1~3.3.13 共 13 张表全部建齐），严格执行 3.4 节 DDL 统一规范
- 涉及表：
  - 基础档案：`sys_customer`, `sys_paper`, `sys_machine`, `sys_warehouse`
  - 加工核心：`biz_process_order`(含 snap_print/snap_finish 快照字段), `biz_original_roll`, `biz_finish_roll`, `biz_process_step`, `biz_process_param`, `biz_finish_original_rel`
  - 出库结算：`biz_delivery_order`, `biz_delivery_detail`, `biz_settle_order`, `biz_settle_detail`, `biz_receive_record`
  - 系统辅助：`sys_operation_log`
- 全局统一字段：`is_deleted`, `create_by`, `update_by`, `create_time`, `update_time`, `version`
- 所有索引、唯一约束一并建立

#### P0-2：基础档案 CRUD API（第1周）
- 客户管理：列表（搜索分页）、新增、修改、删除（软删除）
- 纸张档案：列表、新增、修改
- 机台档案：列表、新增、修改
- 仓库档案：列表、新增、修改
- 统一响应格式：`{ "code": 200, "data": {}, "message": "success" }`

#### P0-3：加工单基础 API（第2周）
- 加工单列表查询（状态筛选、日期范围、客户筛选）
- 创建加工单（含原纸明细录入）
- 加工单详情查询（含原纸、成品、工序）
- 原纸明细增删改
- **状态字段只读，不做状态机流转**（Phase 2 实现）

### 2.2 前端任务

#### P0-4：项目框架初始化（第1周前2天）

- 路由配置：

  - `/work-orders/:id` 加工单详情

#### P0-5：基础档案页面（第1周）

- 纸张/机台/仓库管理页面（简化版表格）

#### P0-6：加工单列表与创建页（第2周）
- 加工单列表页：状态色标（橙/蓝/红/绿/灰）、筛选条件、列配置

## 三、Phase 2：主流程打通期（第3-4周）

### 🎯目标
跑通一张完整单据：创建 → 打印 → 回录 → 已完成 → 出库 → 结算。

### 3.1 后端任务

#### P1-1：状态机实现（第3周前2天）
- 按 V4.1 3.5 节状态迁移矩阵实现
- 加工单状态流转：`待下发→加工中→待回录→已完成→已结算`
- 原纸状态流转：`待加工→加工中→完成/直发/报废`
- 成品状态流转：`待入库→已入库→已出库/报废`
- 状态变更校验，禁止非法跳转

#### P1-2：成品卷号生成（第3周）
- 全局唯一编码：1位大写字母（排除I/O/L/Z）+ 6位数字
- 分布式锁防并发重复
- 支持批量生成、追加备用号、作废封存
- 卷号查重接口

#### P1-3：打印接口（第3周）
- 打印时锁定原纸标称参数
- 生成下发快照（JSON存档）；**`snap_print` 根节点必须写入 `"schema_version":"1.0"`，缺版本号拒绝写入**
- 打印次数记录、补打权限控制
- 加工单状态→加工中
- 验收：`snap_print` 必含 `schema_version`，无版本快照写入被拒绝

#### P1-4：回录接口（第3-4周）
- 原纸实际参数录入（克重、门幅、复称重量）
- 成品实际重量、规格录入
- **重量闭合优先（V4.1 文档 2.6.1-G）**：某件已提供实称重量时该件取实称值，面积分摊仅作用于剩余未实称件，末件走尾差倒挤
- 三级重量闭合校验：
  - ±2% 弱提示
  - ±2%~±5% 强警告（填原因）
  - >±5% 强制拦截（管理员授权）
- 一键重算全部预估数据
- 回录提交：状态→已完成，生成完成快照；**`snap_finish` 根节点必须写入 `"schema_version":"1.0"`，缺版本号拒绝写入**
- 验收①（重量闭合）：成品实称重量 + 工序损耗 + 报废 + 修边 = 原纸实际重量 ±可控误差，末件倒挤完成整卷闭合
- 验收②（快照版本）：`snap_finish` 必含 `schema_version`，无版本快照写入被拒绝

#### P1-5：基础计费（第4周）
- 锯纸费：实际刀数 × 锯纸单价（四舍五入取整）
- 复卷费：原纸实际吨位 × 复卷单价（四舍五入取整）
- 附加费合计

#### P1-6：出库 API（第4周）
- 创建出库单（勾选已入库成品+直发原纸）
- 出库确认（扣减库存，成品状态→已出库）
- 出库单列表
- **直发原纸 source_type=2 三不约束（V4.1 文档 5.5）**：直发记录①不入库（回录即已出库可选状态）②不允许拆分/调重③不进入加工状态机、不可作为其他加工单原卷输入
- 验收：直发记录不入库、不拆分、不进入加工状态机；违反三不约束的操作被拒绝

#### P1-7：结算 API（第4周）
- 按单生成结算单
- 按月度批量合并结算
- 多次收款登记
- 现结客户出库拦截开关

### 3.2 前端任务

#### P1-8：打印功能（第3周）
- 母卷左合并布局打印模板
- 预打印成品卷号、母卷工艺标注
- 混合工艺单据红色大字标识
- 打印前锁定确认弹窗

#### P1-9：回录页面（第3-4周）
- 上下分层对照布局
- 上层：原纸复核区（标称灰色不可编辑 / 实际参数可编辑）
- 下层：成品录入区（与打印单一致的母卷左合并布局）
- 顶部实时重量偏差状态栏（超阈值变色）
- 快捷工具：一键复制标称值、批量重算、±1kg微调、倒挤尾差

#### P1-10：出库与结算页面（第4周）
- 出库单创建（成品勾选表格）
- 结算单生成（费用汇总展示）
- 收款登记弹窗

### 🧩阶段成果
- ✔ 一个单据完整跑通
- ✔ 前后端闭环成立
- ✔ 打印、回录、出库、结算均可操作

## 七、执行约束（必须遵守）

### 7.1 强制规则
- ❌ 禁止前端实现业务判断（金额计算、状态判断、校验规则全部后端返回）
- ✅ 状态必须由后端裁决，前端只做展示和交互
- ✅ 所有写操作必须走 version 乐观锁
- ✅ 所有金额以后端返回为准，前端不做计算
- ❌ 禁止前端自行修改后端返回的金额、状态、重量

### 7.2 技术约束
- 金额：统一 `decimal(12,2)` 或 `decimal(10,2)`，禁止 float/double
- 重量：统一 `decimal(10,3)`，精度到克
- 状态：统一 `tinyint`
- 主键：统一 `varchar(36)` UUID
- 软删除：统一 `is_deleted tinyint NOT NULL DEFAULT 0`
- 后端技术栈：**SpringBoot（已定稿）**，AOP 字段审计、Redisson 分布式锁（成品卷号防并发）、声明式事务保障回录/出库一致性

## 八、风险控制

### 高风险点
| 风险                 | 影响         | 对策                       |
| -------------------- | ------------ | -------------------------- |
| 成品卷号生成并发冲突 | 卷号重复     | Phase 2 即加分布式锁       |
| 回录重量校验过严     | 业务无法提交 | 三级分级管控，管理员可放行 |
| 混合工艺计费错误     | 结算金额不准 | Phase 3 重点测试           |
| 状态机实现分散       | 状态跳转混乱 | 统一 Service 层状态机      |

### 对策
- Phase 2 之前禁止优化
- Phase 1 只允许"能跑"
- Phase 3 再收敛规则
- Phase 4 才做工程优化

---

## 九、最终总结

**这个项目不是"做完"，而是"分四次收敛复杂度"。**

- Phase 1：骨架能跑
- Phase 2：主流程通
- Phase 3：规则稳
- Phase 4：可上线

**每一步都只做当前阶段必要的事，不提前引入下一阶段的复杂度。**
