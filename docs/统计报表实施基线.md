# 统计报表实施基线

## 目的

本文件冻结统计报表 V1 的现有计算语义，并定义 V2 实施期间的黄金案例与节点审查门禁。任何节点只有在对应口径、接口、权限、测试和 UI 审查全部通过后才能进入下一节点。

## 当前执行事实

- 页面总览、维度、明细和导出均读取 `ReportMapper` 实时 SQL。
- 默认只统计状态为已完成或已结算的加工单。
- 当前日期筛选统一使用加工单 `accounting_date`。
- 收款、现金和废纸抵扣先按结算单分摊到加工单，再按加工单归属日期展示。
- `metricReleaseUuid` 已进入查询 DTO，但尚未参与 `ReportMapper` 的计算分支。
- 物化指标值和报表快照已有存储结构，但尚未成为页面查询读取源。

## 黄金案例

| 编号 | 场景 | 必须验证的结果 |
|---|---|---|
| `G01` | 单一锯纸加工单 | 数量、重量、刀数、锯纸费正确 |
| `G02` | 单一复卷加工单 | 复卷费、吨位、损耗正确 |
| `G03` | 同单包含锯纸和复卷 | 工序费用不重复、不遗漏 |
| `G04` | 同单包含多个品名 | 加工单去重，产品维度分摊一致 |
| `G05` | 多原卷产出多成品 | 原卷、成品数量和重量正确 |
| `G06` | 直发原卷 | 不计生产原卷和加工损耗 |
| `G07` | 余料和中间产物 | 只统计有效最终成品 |
| `G08` | 成品作废或卷号作废 | 不进入有效成品指标 |
| `G09` | 加工单无原纸重量 | 损耗率为零且不得除零 |
| `G10` | 成品重量大于原纸重量 | 质量异常可追溯，不生成负损耗 |
| `G11` | 未结算加工单 | 全额进入待结算应收 |
| `G12` | 部分加工单进入结算 | 已结算和待结算金额边界正确 |
| `G13` | 结算单部分收款 | 已收与未收金额正确 |
| `G14` | 现金加废纸抵扣 | 收款构成相加等于有效已收 |
| `G15` | 作废结算单 | 不覆盖应收，不阻止后续结算 |
| `G16` | 作废收款流水 | 不计入已收和现金流 |
| `G17` | 6 月加工、7 月收款 | 应收视图归属 6 月，现金流视图归属 7 月 |
| `G18` | 免费加工或零加工费 | 排行仍使用统一指标与单位 |
| `G19` | 月份无业务 | 时序返回零值，不与缺失数据混淆 |
| `G20` | 物化分区缺失 | 明确返回未覆盖或实时回退，不得静默为零 |
| `G21` | 固定历史指标版本 | 页面、导出、订阅和预警结果一致 |
| `G22` | 用户无财务权限 | 页面、接口、导出均不返回财务指标 |
| `G23` | 客户数量超过 200 | 远程候选仍能检索并回显目标客户 |
| `G24` | 复杂查询超过成本阈值 | 转异步任务且保留完整查询摘要 |

## 节点审查门禁

每个实施节点依次执行以下检查，任一失败必须修复后重新从对应检查开始：

1. 数据口径：指标粒度、时间轴、单位、聚合规则没有歧义。
2. 数据库：迁移可重复执行，约束、索引、回填和降级路径明确。
3. 后端：参数校验、权限、幂等、并发、异常边界和查询成本受控。
4. API：前后端字段、枚举、空值、分页、排序和错误码一致。
5. 前端：加载、刷新、空、错、部分数据、无权限状态完整。
6. UI/UX：1366、1920、2560 宽度无溢出、重叠、断字和布局抖动。
7. 测试：单元、契约、集成、端到端及关键截图通过。
8. 性能：标准查询、复杂查询、导出和回填达到对应阈值。
9. 安全：行级、字段级、导出和缓存权限一致。
10. 回归：旧模块未被意外改变，差异都有明确业务解释。
