# 卷筒纸加工管理系统 开发文档 V4.1

## 版本变更说明
1. V4.0 基础框架：成品全局唯一字母+6位数字卷号、母卷合并打印、出库结算并行解耦、车间现场复核回录、单卷三级加工模式
2. V4.1 核心修正：解决同单据混合锯纸/复卷工艺问题，废弃单据顶层单一工艺字段，工艺下沉至单卷工序维度；新增混合工艺标识字段；全文档同步修正工艺相关逻辑、表结构、页面、打印、统计规则
3. 所有需求全部落地：批量导入、破损多图上传、现场冗余备用卷号、次结/月结双结算管控、无标签车间手写作业、原卷标称/实际双参数、多工序追加、重量三级校验、件件溯源管控

## 一、系统概述
### 1.1 系统定位与业务目标
面向卷筒纸锯切、复卷来料加工企业，打造办公室制单+车间纸质流转+线上回录+库存出库+结算收款全链路数字化系统。
核心解决行业痛点：来料参数不标准、水湿破损现场灵活裁切、成品需单件管控、同一单据多种加工工艺、先提货/先结账两种合作模式、手工计算损耗与费用误差大、对账溯源繁琐。
业务目标：
1. 单卷独立配置加工模式，同单混合标准加工/现场定尺/不加工直发，无需拆单
2. 支持同单据多母卷分别做锯纸、复卷，单母卷可先锯后复卷追加工序
3. 成品全局永久唯一编码，车间手写抄号即可实现单件库存、出库、结算溯源
4. 车间线下复核原纸克重/门幅/重量，办公室回录修正，标称、实际数据双留存用于对账
5. 出库、结算无强制先后顺序，适配款到发货、月度结账两类客户
6. 自动完成重量预估、损耗计算、分工艺计费，金额四舍五入取整
7. 完整图片存档、操作日志、双版本快照，全链路可追溯

### 1.2 适用业务场景
1. 标准加工：来料参数准确，开单预设成品规格，车间按单生产
2. 现场定尺加工：水湿、破损原纸，裁切规格现场确定，开单仅预分配成品卷号
3. 混合直发加工：同批来料部分加工、部分原纸直接出库，统一一张单据管理
4. 混合工艺加工：一张单据内部分母卷锯纸、部分复卷；单卷先锯切除水湿再复卷
5. 乱码纸无原始卷号：系统自动生成全局成品编号，不受母卷编号限制
6. 两类结算场景：次结客户结清费用方可提货；月结客户先出库月底统一对账

### 1.3 核心设计原则
1. 人工优先：系统仅做计算参考，所有重量、规格、费用均可人工修改，修改强制留痕
2. 颗粒度最小化：加工模式、工艺、参数全部下沉至单卷/单工序，单据不再全局统一约束
3. 贴合线下作业：以纸质加工单为流转核心，车间全程手写，无打印标签流程依赖
4. 灵活管控：重量误差分级预警、现结出库可配置拦截开关、作废卷号永久封存不复用
5. 数据安全：软删除、双版本快照、字段级操作日志、成品卷号全局唯一索引兜底
6. 扩展预留：所有核心表预留扩展字段，新增工艺、报表、客户模板无需大规模改表

## 二、完整业务主流程
### 2.1 全链路总流程
```
基础档案录入（客户/纸张/机台/仓库）
        ↓
新建加工单 → 选择客户（支持弹窗快捷新增客户）→ 批量导入/手动录入原卷标称参数
        ↓
逐行设置每卷原纸加工模式（标准加工/现场定尺/直发）+ 为本卷选择主工艺（锯纸/复卷）
        ↓
标准加工卷录入成品规格，系统自动生成全局唯一成品卷号；支持手动追加备用卷号
        ↓
打印纸质加工单（母卷左合并布局，预打印成品卷号、母卷工艺标注）→ 单据状态：加工中
        ↓
车间线下作业：
  1. 逐卷复核：称重、量门幅、实测克重，手写实际参数在单据复核栏
  2. 按预打印成品卷号加工，将对应卷号抄写在纸卷端面
  3. 有需要追加工序、多分卷、报废，手写在对应成品行
        ↓
纸质单据回收，标记单据状态：待回录
        ↓
办公室两步回录：
  ① 录入车间实测原纸实际克重/门幅/重量，系统一键重算全部预估数据
  ② 对照纸质卷号录入成品实际重量、尺寸、追加工序，作废未使用备用卷号
        ↓
系统三级重量闭合校验，确认提交 → 单据状态：已完成，加工成品、直发原纸自动入库
        ↓
【出库、结算并行无先后强制约束】
┌─────────────┴─────────────┐
↓                           ↓
创建出库单                   生成结算单
勾选已入库成品（加工成品+直发原纸） 录入开票信息，系统分工艺自动计算加工费
配置现结拦截管控开关        支持多次收款、分批登记
扣减成品库存，记录提货信息  支持按单结算/月度批量合并结算
└─────────────┬─────────────┘
              单据归档，全数据快照留存
```

### 2.2 两类结算分支流程
#### 分支1：次结现结客户（先结清再出库）
加工单已完成 → 生成结算单对账 → 客户付款，完成收款登记 → 方可创建出库单
系统配置项：未结清时出库弹窗警告 / 强制拦截

#### 分支2：月结客户（先出库后对账）
加工单已完成 → 自由创建出库单完成提货；每月底筛选客户全部已完成单据，批量合并生成月度结算单统一对账收款

### 2.3 单据完整状态流转
1. 加工单
- 待下发：新建录入完成，未打印，可全量编辑
- 加工中：纸质单打印下发，原纸标称参数锁定，仅可标记待回录
- 待回录：车间单据回收，开放实际参数、成品、工序编辑权限
- 已完成：回录校验提交，成品入库，可出库、可结算
- 已结算：绑定结算单，加工费用字段锁定不可修改
2. 单卷原纸状态：待加工→加工中→加工完成/直发/报废
3. 成品状态：待入库→已入库→已出库/报废；卷号状态：预生成/已使用/作废
4. 出库单：待出库 → 已出库签收
5. 结算单：待结算 → 部分收款 → 全部结清

### 2.4 混合工艺两种场景业务逻辑
#### 场景A：同单不同母卷工艺不同
原卷1主工艺锯纸，原卷2主工艺复卷；系统自动标记单据`is_mix_process=1`；打印单每个母卷区块单独标注加工工艺；结算分开统计锯纸刀数费用、复卷吨位费用。

#### 场景B：同一母卷多工序叠加（主工艺+追加工序）
母卷主工艺复卷，车间追加锯切水湿边工序；工序表新增两条记录：主工序复卷、追加工序锯纸；单据标记混合工艺，两道工序分别计费叠加。

### 2.5 锯纸工艺流程
1. 单卷选择主工艺锯纸，录入成品门幅，系统生成成品行+全局卷号
2. 自动计算总修边宽度、理论刀数（成品数量-1）、预估成品重量
3. 支持手动修改实际总刀数（包含切头、切两边损耗刀），计费以实际刀数为准
4. 最后一件成品支持一键重量倒挤，消除计算尾差
5. 加工费公式：单卷锯纸费 = 实际刀数 × 锯纸单价，结果四舍五入取整

### 2.6 复卷工艺5类标准计算模式
所有直径、纸芯直径录入单位为英寸，系统自动换算面积做重量分摊：
1. 改门幅不变直径：按门幅比例分摊扣除修边重量
2. 改直径不变门幅：按纸卷横截面积占比分摊重量
3. 同时改直径+改门幅：先按直径分层分摊，再按门幅折算单件重量
4. 内外层分层不同规格：多层面积独立计算，每层分配对应成品规格
5. 多母卷合并复卷：勾选多条原卷，自定义重量分摊比例生成成品
复卷加工费公式：单卷复卷费 = 原纸实际吨位 × 复卷单价，四舍五入取整

> **算法基准说明**：上述5类模式的精确公式、换算系数、修边分摊、尾差倒挤及完整用例，统一在 **2.6.1《复卷重量分摊计算规格书》** 中定义，作为后端编码与测试验收的唯一基准；该规格书为开工前置交付物，**必须在 Phase 1 完成**。

#### 2.6.1 复卷重量分摊计算规格书（Phase 1 前置交付）
本规格书是全系统计算精度的命脉，所有重量分摊、修边扣减、尾差消除均以此为准。

**A. 基础换算与符号约定**
- 英寸→毫米换算系数：`1 英寸 = 25.4 mm`（直径、纸芯直径录入为英寸，内部统一换算为 mm 后参与面积计算）
- 符号：`W_actual` 原纸实际重量(kg，取 `biz_original_roll.actual_weight`)；`D_out` 成品外径(mm)；`D_core` 成品纸芯直径(mm)；`width` 门幅(mm)；`N` 成品件数

**B. 纸卷横截面积公式**
```
纸卷横截面积 S = π × ( (D_out/2)² − (D_core/2)² )      单位 mm²
```
当模式只改门幅、外径与纸芯一致时，各件 `S` 相等，面积占比退化为按门幅比例分摊（即模式1）。

**C. 单件重量分摊公式（通用式，适用模式1~4）**
```
总面积 S_total = Σ 各成品件面积 S_i
单件理论重量 weight_i = W_actual × ( S_i / S_total ) − 本件修边重量 trim_weight_share_i
```
- 模式1（改门幅不变直径）：`S_i ∝ width_i`，面积占比即门幅占比
- 模式2（改直径不变门幅）：`S_i` 按 B 式横截面积计算，门幅相同
- 模式3（改直径+改门幅）：先按外径/纸芯算横截面积，再乘以各件门幅占比，即 `S_i = 横截面积_i × (width_i / 原纸门幅)`
- 模式4（内外层分层不同规格）：每一层独立按 B 式计算面积并参与 `S_total`，分层参数存 `biz_process_param`
- 模式5（多母卷合并复卷）：不按面积，按用户自定义分摊比例 `ratio_i`（Σ ratio_i = 1）分摊，`weight_i = (Σ各母卷 W_actual) × ratio_i − trim_weight_share_i`

**D. 修边分摊公式**
```
总修边重量 trim_weight_total = ( 总修边宽度 / 原纸门幅 ) × W_actual
单件分摊修边 trim_weight_share = trim_weight_total / N
```
（总修边宽度 ≤ 1mm 时执行 2.8 修边豁免，trim_weight_total 一键清零）

**E. 尾差倒挤规则（消除小数累计误差）**
```
最后一件成品重量 weight_N = W_actual − Σ(前 N−1 件 weight_i) − 总损耗重量 total_loss_weight
```
最后一件不再走 C 式分摊，直接由整卷余量倒挤，保证整卷重量严格闭合（对接 5.7 三级校验）。

**F. 完整用例（含手工计算期望值，作为单元测试基准）**

*用例1 — 模式2 改直径不变门幅：*
- 原纸：`W_actual = 1000.000 kg`，门幅 1200mm，原纸外径 40 英寸=1016mm，纸芯 3 英寸=76.2mm
- 拆 2 件成品，均不变门幅：成品A 外径 30 英寸=762mm、成品B 外径 28.28 英寸≈718.3mm（示例），纸芯均 76.2mm
- 面积：`S_A = π×((762/2)²−(76.2/2)²)=π×(145161−1451.61)=π×143709.39 ≈ 451471 mm²`；`S_B = π×((718.3/2)²−(38.1)²)=π×(128988.6−1451.61)=π×127537 ≈ 400666 mm²`
- `S_total ≈ 852137`；无修边、无损耗
- `weight_A = 1000 × (451471/852137) ≈ 529.81 kg`
- `weight_B`（最后一件，倒挤）`= 1000 − 529.81 − 0 = 470.19 kg`
- 校验：529.81 + 470.19 = 1000.00 ✔ 整卷闭合

*用例2 — 模式1 改门幅不变直径 + 修边：*
- 原纸：`W_actual = 800.000 kg`，门幅 1500mm；拆 3 件，门幅分别 500/500/480mm，总修边宽度 20mm
- 总修边重量 `trim_weight_total = (20/1500)×800 = 10.667 kg`；单件修边 `= 10.667/3 ≈ 3.556 kg`
- 面积占比按门幅：`S_total(门幅) = 500+500+480 = 1480`（修边 20mm 不计入成品门幅）
- `weight_1 = 800×(500/1480) − 3.556 = 270.270 − 3.556 = 266.714 kg`
- `weight_2 = 800×(500/1480) − 3.556 = 266.714 kg`
- `weight_3`（最后一件，倒挤，总损耗以修边总重计）`= 800 − 266.714 − 266.714 − 10.667 = 255.905 kg`
- 校验：266.714 + 266.714 + 255.905 + 10.667(修边损耗) = 800.000 ✔ 整卷闭合

**G. 重量闭合优先原则（实称值优先于公式）**
当车间回录已提供某件成品的实称重量（`biz_finish_roll.actual_weight` 非空）时：
1. 该件**不再走 C 式面积分摊**，直接取其实称值参与整卷重量；
2. 面积分摊仅作为**剩余未实称件**的分配权重（即 `S_total` 仅累计未实称件面积，`W_actual` 先扣除已实称件重量后再分摊）；
3. 最后一件仍执行 E 式尾差倒挤，**以原纸实际总重为唯一闭合基准**，吸收全部计算尾差。
此原则保证：人工实测数据永远优先于系统理论估算，系统估算只填补尚未实测的空缺。

### 2.7 现场追加工序流程
1. 开单仅配置主工艺，下发车间后现场临时增加工序
2. 车间纸质单手写工序类型、刀数/加工吨位、损耗重量
3. 回录页面工序Tab新增工序行，标记为追加工序
4. 每道工序独立计算费用，全部取整后累加为本卷总加工费
5. 各工序损耗重量自动汇总，参与整卷重量闭合校验

### 2.8 异常处理流程
1. 原纸参数修正：仅回录阶段修改实际克重/门幅/复称重量，标称参数永久留存做对比溯源
2. 人工调整成品重量：专属调重按钮，强制填写调整原因，单元格高亮标记、日志完整记录
3. 修边豁免：单卷总修边宽度≤1mm可一键清零修边重量，同步重算成品预估
4. 分批出库：同加工单成品支持多次提货，出库单仅勾选对应成品，库存分次扣减
5. 重量超差管控：按偏差比例分三级提示/拦截，超阈值需填写异常原因或管理员授权

### 5.5 出库管理模块
1. 出库单独立单据，与加工单完全解耦，加工单标记已完成即可创建出库，不受结算状态限制
2. 选客户后自动筛选该客户全部未出库成品，同时展示加工成品、直发原纸，自由勾选合并提货
   > **直发原纸出库落点（方案A，已定稿）**：直发原纸（`biz_original_roll.process_mode=3`）在回录提交时，系统自动为其生成一条 `source_type=2` 的成品记录写入 `biz_finish_roll`，其 `finish_roll_no` 直接沿用来料母卷号（不占用全局字母流水）。由此出库统一从 `biz_finish_roll` 勾选，`biz_delivery_detail` 仅需关联成品表一张，无需双表关联。
   >
   > **source_type=2 三不约束（强制）**：① **不允许入库**——回录生成即直接置为已出库待选状态（`finish_status` 跳过待入库，按已入库可出库处理，不进入加工成品入库流程）；② **不允许拆分**——单件即整体，禁止再分卷、禁止调重拆分；③ **不允许进入后续加工流程**——该记录不可作为任何其他加工单的原卷输入（`biz_original_roll`）来源。
3. 系统配置开关：次结客户未结清全部款项时，创建出库弹窗警告 / 强制拦截保存
4. 支持单次单据分多批出库，多张加工单成品合并一张出库单
5. 出库记录提货人、车牌号、柜号、签收信息；出库提交自动扣减成品库存，成品状态改为已出库

### 5.6 结算收款模块
1. 两种生成模式：按单生成结算、按月批量汇总客户全部已完成单据合并结算
2. 自动区分锯纸加工费、复卷加工费分开汇总展示，每道工序费用独立四舍五入取整后累加
3. 支持开票/不开票切换，自动计算不含税金额、税额
4. 一笔结算单支持多次收款登记，记录收款方式（现金/转账/微信/支付宝）、收款时间、金额
5. 自动生成客户对账单Excel，包含原纸明细、成品明细、分工艺费用、出库状态、已收/待收金额

### 5.7 重量三级闭合校验（基准：车间回录实际复称重量）
校验以单卷原纸为单位，闭合公式与取数口径如下（精确到 0.001kg）：
```
闭合公式：
原纸实际重量(actual_weight) ≈ Σ成品实际重量 + Σ工序损耗 + Σ报废重量 + Σ修边重量

偏差率 = |原纸实际重量 − 右侧合计| / 原纸实际重量 × 100%
```
各项取数口径（均限定为当前原卷来源）：
- 原纸实际重量：`biz_original_roll.actual_weight`（当前原卷）
- Σ成品实际重量：`SUM(biz_finish_roll.actual_weight)`，来源为当前原卷的全部成品
- Σ工序损耗：`SUM(biz_process_step.loss_weight) WHERE original_uuid = 当前原卷`
- Σ报废重量：`SUM(biz_finish_roll.scrap_weight)`，来源为当前原卷
- Σ修边重量：`SUM(biz_finish_roll.trim_weight_share)`，来源为当前原卷

分级管控（基于上述偏差率）：
1. 一级弱提示：偏差±2%以内，浅色提示条，直接保存无拦截
2. 二级强警告：偏差±2% ~ ±5%，弹窗确认，必须填写异常原因才可提交
3. 三级强制拦截：偏差＞±5%，默认禁止保存；特殊业务场景管理员授权后可放行，强制记录详细说明
   > **授权放行强制留痕**：任何超差授权放行操作必须写入 `sys_operation_log`，其中 `action_type` 固定为「超差放行」，`remark` 必填放行原因（不可为空），`operator`（操作人）不可为空。未生成该日志的放行视为无效，禁止提交。

## 六、全链路溯源与数据快照
### 6.1 双节点永久快照（不可修改）
1. 下发快照：打印加工单瞬间，保存全部原纸标称参数、预生成成品卷号、预估重量、预设工艺，用于和车间纸质单据对账。**存储于 `biz_process_order.snap_print`（JSON），无需独立快照表。**
2. 完成快照：回录校验提交后，保存车间实测实际参数、成品真实重量、全部工序费用、作废卷号清单，作为结算、出库、统计唯一基准数据。**存储于 `biz_process_order.snap_finish`（JSON）。**
支持一键对比 `snap_print` 与 `snap_finish` 两份快照差异，清晰展示车间修改、损耗、分卷变动。
> **快照结构版本**：两份快照 JSON 根节点均必须包含 `"schema_version": "1.0"` 字段。后续表结构或字段语义变更时递增版本号（1.0→1.1→…），保证历史快照按其所属版本可解析、可追溯，新旧快照不混淆。

### 6.2 全维度操作留痕
1. 字段级日志：所有数值、文本、状态修改自动记录：字段名、修改前值、修改后值、操作人、操作时间
2. 关键动作日志：打印、回录、结算、收款、作废卷号、批量删除全部独立记录
3. 图片溯源：破损图片绑定对应原纸UUID，永久存档可随时查看
4. 打印日志：记录每张单据打印次数、操作员、打印时间、是否补打

### 6.3 双向穿透溯源
1. 正向溯源：原纸 → 对应全部成品（卷号、规格、重量）→ 成品对应出库单、结算单、收款记录
2. 反向溯源：结算单/出库单 → 关联加工单 → 单卷原纸、工序、实测参数、破损图片、车间手写记录

## 七、技术实现方案
### 7.1 技术栈
- 前端：React18 + Ant Design 5 + ProTable、ExcelJS导入导出、本地文件图片存储
- 后端：**SpringBoot（已定稿，不再二选一）**，RESTful标准接口
  - 选型理由：① AOP 日志切面成熟，可零侵入实现字段级审计；② 分布式锁（Redisson）开箱即用，满足成品卷号并发生成；③ 声明式事务管理可靠，保障回录提交、出库扣减的一致性；④ 与本文档后端机制描述（AOP切面、分布式锁、独立计算引擎）的 Java 生态契合。
- 数据库：MySQL8.0，支持JSON字段、唯一索引、事务、乐观锁
- 部署：本地内网服务器部署，数据本地存储，不上公网

### 7.2 后端核心机制
1. 全局软删除自动过滤，业务代码无需额外判断
2. AOP切面自动生成字段修改日志，无业务代码侵入
3. 成品卷号生成加分布式锁，多账号同时开单绝对不重复
4. 工艺计算引擎独立工具类封装，后续新增工艺仅扩展方法，不改动主流程
5. 快照自动生成工具，打印、回录完成自动归档完整JSON快照数据

### 7.3 前端核心机制
1. 通用封装业务表格组件，统一拖拽、行编辑、高级查询、批量操作
2. 表单动态渲染：根据单卷工艺自动切换锯纸/复卷参数输入面板
3. 数值字段实时联动计算，重量、费用修改即时刷新汇总卡片
4. 本地localStorage存储用户表格布局、自定义查询方案
