# 加工配置与复卷模型重构方案

> 目的：把加工单配置从“能填规格、能保存”重构为“好用、人性化、方便”的业务工作台。本文基于 V4.1 文档、现有整改过程，以及本轮用户对复卷 5 类、批量配置和详情展示的补充说明。

## 一、结论

当前加工单模块的主要问题不是单点 bug，而是业务模型偏差：系统把母卷加工简化成 `finishWidth + finishDiameter + count` 的扁平成品规格表，无法表达真实复卷中的“直径分段、门幅排布、修边、跨母卷接纸、批量配置、母卷产出视图”。

后续整改不应继续追求最小改动，应围绕“母卷加工方案”做结构性重构：

```text
母卷 → 加工方案 → 直径分段 → 门幅排布 → 成品生成 → 来源关系 → 打印/回录/计费
```

前端验收标准必须从“能用”提升为“好用、人性化、方便”。如果为了好用需要增加后端字段、接口、DTO 或数据表，应主动改后端，不应为了少改代码牺牲体验。

## 二、当前主要问题

### 2.1 复卷模式理解错误

当前系统将成品规格表达为：

```text
finishWidth
finishDiameter
finishCoreDiameter
count
```

并把 `count` 参与门幅横向累加，例如 `finishWidth * count`。这导致以下真实业务无法表达：

```text
原卷门幅 1000
成品 950 × 2
每件各有 50mm 修边
```

系统会误解为：

```text
950 + 950 横向排布，总门幅 1900，超过 1000
```

真实含义应是：

```text
两个直径/长度方向成品段，每段门幅 950，每段修边 50
```

根因：系统混淆了“横向门幅排布数量”和“直径/长度方向分卷数量”。

### 2.2 复卷 5 类只做了名称，没有建立真实模型

复卷 5 类应按以下业务含义重建：

1. 改门幅不变直径：只改变门幅排布，不切直径段。
2. 改直径不变门幅：门幅保持母卷门幅，只沿直径/长度方向分成多个小卷。
3. 改门幅 + 改直径：同一门幅排布在多个直径段上重复。
4. 内外层分层：不同直径段使用不同门幅排布。
5. 多母卷合并复卷：多个母卷按顺序接纸，一个成品可能跨多个母卷来源。

当前实现把模式 4 简化为“层面积参数”，把模式 5 简化为“来源比例”，都不足以支撑真实生产。

### 2.3 前端配置流程不符合业务操作习惯

真实业务需要：

```text
勾选一批母卷 → 设置同样加工方式/规格 → 批量应用 → 个别微调
```

当前页面是：

```text
左侧逐个点母卷 → 右侧单卷配置 → 粗略复制到选中母卷
```

问题包括：

- 不方便批量配置同规格母卷。
- 复卷配置没有“方案编辑器”和“成品预览”。
- 用户保存前看不清会生成什么成品、多少修边、什么重量。
- 多母卷合并无法直观看到接纸关系和来源关系。

### 2.4 详情页不是业务视角

用户希望详情一行一行看到：

```text
哪个母卷 → 什么加工方式 → 做成什么规格 → 生成哪些成品
```

当前详情更偏数据库实体展示：原纸表、成品表、工序表分散显示，不能快速回答业务问题。

### 2.5 后端保存模型不足

现有 `FinishConfigSaveDTO`、`FinishConfigSpecDTO` 只能承载扁平规格，不能表达：

- 直径分段。
- 每段门幅排布。
- 每段占比或目标直径。
- 修边项。
- 跨母卷接纸。
- 多母卷消耗顺序。
- 一个成品对应多个来源母卷。

`biz_process_param` 和 `biz_finish_original_rel` 可以保留，但需要围绕新的加工方案模型重新组织。

## 三、目标体验

### 3.1 配置页目标

新配置页应是“母卷加工配置工作台”，而不是单卷表单。

推荐布局：

```text
左侧：母卷列表
- 多选
- 全选同规格
- 已配置/未配置状态
- 加工方案摘要

右侧：加工方案编辑器
- 加工模式
- 主工艺
- 复卷模式
- 按模式展示不同编辑器

下方或侧栏：实时预览
- 预计成品组成
- 修边
- 预计重量
- 来源母卷
- 应用到选中母卷
- 保存当前/保存全部
```

### 3.2 批量配置目标

应支持：

- 勾选多个母卷后统一配置加工方式和规格。
- 按母卷规格快速筛选/全选。
- 将当前方案应用到选中母卷。
- 将常用方案保存为模板。
- 允许批量应用后单卷微调。

### 3.3 详情页目标

详情页增加“母卷产出视图”：

```text
母卷号 | 原纸规格 | 加工模式 | 主工艺 | 方案摘要 | 产出成品 | 修边 | 来源关系 | 状态
```

展开后显示：

```text
分段明细
成品卷号
每个成品来源母卷
修边/报废/预估重量
```

多母卷合并时按组展示：

```text
A001 + A002 → 500 × 6
成品 3：A001 60% + A002 40%
```

## 四、复卷业务模型

### 4.1 基础概念

复卷模型应由两个核心维度组成：

```text
直径/长度方向分段 segments × 每段门幅排布 layoutItems
```

每个 `segment` 表示母卷直径/长度方向的一段，每段内有一组门幅排布。

### 4.2 模式 1：改门幅不变直径

只配置一段，段内配置门幅排布。

示例：

```text
母卷 1000
排布：500 + 500
成品：500 × 2
修边：0
```

示例：

```text
母卷 1000
排布：950
修边：50
成品：950 × 1
```

### 4.3 模式 2：改直径不变门幅

门幅固定为母卷门幅，只配置直径分段。

示例：

```text
母卷 1000
直径分 3 段
成品：1000 × 3
```

### 4.4 模式 3：改门幅 + 改直径

配置一套门幅排布，并在多个直径段上重复。

示例：

```text
母卷 1000
排布：500 + 500
直径分 2 段
成品：500 × 4
```

示例：

```text
母卷 1000
排布：950 + 50修边
直径分 2 段
成品：950 × 2
修边：50 × 2
```

### 4.5 模式 4：内外层分层

每个直径段可配置不同门幅排布。

示例：

```text
前 1/3：400 + 600
中 1/3：500 + 500
后 1/3：300 + 300 + 300 + 100
```

成品：

```text
400 × 1
600 × 1
500 × 2
300 × 3
100 × 1
```

模式 4 是模式 3 的增强版：模式 3 每段排布相同，模式 4 每段排布可不同。

### 4.6 模式 5：多母卷合并复卷

模式 5 不是简单比例分摊，而是在复卷过程中允许多个母卷按顺序接纸。

需要表达：

- 母卷使用顺序。
- 每个母卷消耗比例/重量/长度。
- 某个成品是否跨母卷。
- 每个成品对应哪些来源母卷。
- 来源比例或来源重量。

示例：

```text
母卷 A、B，门幅均 1000
目标排布：500 + 500
母卷 A 做到一部分后用完
成品中间接上母卷 B 继续做到目标直径
最终产出：500 × 6
```

## 五、建议后端模型

### 5.1 新 DTO

建议新增：

```text
FinishConfigPlanDTO
RewindSegmentDTO
RewindLayoutItemDTO
RewindSourcePlanDTO
FinishPreviewVO
```

示例结构：

```json
{
  "processMode": 1,
  "mainStepType": 2,
  "rewindMode": 4,
  "spareCount": 0,
  "segments": [
    {
      "segmentSort": 1,
      "segmentRatio": 33.33,
      "targetDiameter": 30,
      "layoutItems": [
        { "finishWidth": 400, "quantity": 1, "itemType": "FINISH" },
        { "finishWidth": 600, "quantity": 1, "itemType": "FINISH" }
      ]
    }
  ],
  "sourcePlan": []
}
```

`itemType` 建议支持：

```text
FINISH：成品
TRIM：修边
```

### 5.2 新表或结构化存储

推荐新增结构化表，而不是只把 JSON 塞进 `param_json`：

```text
biz_process_plan
biz_rewind_segment
biz_rewind_layout_item
biz_rewind_source_plan
```

如果担心一次改动过大，可第一阶段先结构化 DTO + JSON 落库，第二阶段再拆表。但从详情、打印、回录、统计角度看，结构化表更合适。

### 5.3 后端职责

后端应负责：

1. 校验加工方案是否合法。
2. 根据方案生成成品预览。
3. 根据方案生成真实成品卷号。
4. 计算每件成品预估重量。
5. 计算修边宽度和修边重量。
6. 写入成品来源关系。
7. 写入打印和回录所需快照。

前端不应自己决定最终成品明细，只负责编辑方案和展示后端预览。

5. 详情页以业务语言展示，不以数据库字段展示。

## 七、实施路线

### 阶段 R1：文档和模型确认

1. 更新整改计划，标记当前复卷 5 类基础实现存在模型偏差。
2. 完成本重构方案文档。
3. 明确 5 类复卷输入、输出、修边、重量、来源关系规则。

验收：文档能解释用户提出的 950×2、500+500×2段、内外层分层、多母卷接纸场景。

### 阶段 R2：后端预览模型

1. 新增方案 DTO。
2. 新增成品预览接口。
3. 后端根据方案生成预览，不落正式成品号。
4. 覆盖模式 1、2、3 的单元测试。

验收：输入方案后，后端返回成品组成、修边、预估重量。

### 阶段 R3：前端工作台雏形

1. 重做配置页布局。
2. 支持母卷多选和批量应用。
3. 支持模式 1、2、3 的方案编辑和预览。
4. 保存仍可先接现有接口或临时预览接口，但不能制造假卷号。

验收：用户可以方便配置多个同规格母卷，并在保存前看清成品预览。

### 阶段 R4：模式 4 与模式 5

1. 模式 4 支持多段不同门幅排布。
2. 模式 5 支持多母卷顺序、接纸来源、成品来源关系。
3. 后端保存结构化方案。

验收：能表达用户补充的内外层分层和多母卷接纸案例。

### 阶段 R5：真实保存与详情页

1. 保存方案并生成真实成品卷号。
2. 详情页增加母卷产出视图。
3. 打印快照、回录闭合、计费基于新方案校准。

验收：详情中可一行一行看到每个母卷做成什么规格、产出哪些成品。

## 九、后续每轮推进要求

每一轮实现后必须先审查，再进入下一轮。审查内容包括：

1. 代码是否符合新的业务模型。
2. 前端是否真的更好用，而不是只是能用。
3. 后端字段和 DTO 是否能表达业务含义。
4. 成品生成、修边、来源关系、重量是否闭环。
5. 是否存在历史数据或边界场景遗漏。
6. 后端编译和前端构建是否通过。

未经审查确认，不推进下一阶段。
